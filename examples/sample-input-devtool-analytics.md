# Sample Input: Developer Analytics Tool

We are building a hosted analytics dashboard for open-source maintainers. It pulls GitHub stars, issues, discussions, package downloads, docs traffic, and community mentions into one adoption report.

Current stage: first users. Five open-source maintainers have tried a prototype for free.

Evidence:

- 5 maintainers used the prototype.
- 3 said they need adoption metrics for sponsors or internal stakeholders.
- 2 maintainers said they already use GitHub Insights and npm download stats manually.
- No one has paid yet.
- One maintainer asked whether the report can be exported as a monthly PDF.

Candidate segments:

1. Independent open-source maintainers looking for sponsors.
2. DevRel teams at developer-tool startups.
3. Engineering managers maintaining company-backed open-source projects.

Constraints:

- Solo builder.
- Strong GitHub and developer community access.
- No sales team.
- Can write technical content and ship quickly.

Decision needed:

Choose a first beachhead and GTM motion. Avoid a generic marketing plan.
